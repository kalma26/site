Phase I — Character.edf deep-dive tutorial series

Master Character.edf with a focused, practical walkthrough. Each topic includes tips to avoid common pitfalls.

Understanding each sheet
Action: Action states, animation hooks, timing windows.
Animus: Summon-related parameters and restrictions.
Class: Class IDs, stat baselines, role flags.
Exp: Level thresholds and curve definitions.
Grade: Rank progression and perks.
MasteryLimData: Caps for skills and equipment proficiencies.
MonsterCharacter: Monster variants tied to character schema.
Stat tables, growth curves, and race limitations
  • Baselines: Keep early levels forgiving; avoid steep early spikes.
  • Growth: Use smooth, monotonic progressions; watch secondary stats.
  • Race rules: Respect unique caps and passives when rebalancing.
Editing EXP curve safely
  • Version: Duplicate files and tag with date + change note.
  • Delta-first: Edit small increments; test midgame levels.
  • Cross-check: Ensure quests, drops, and map difficulty align.
Rebalancing classes & monsters
  • Role clarity: Define each class’s niche before editing numbers.
  • Counterplay: Keep at least two soft counters per dominant role.
  • Trial builds: Sandbox with 5–10% adjustments; avoid compounding buffs.
Common bugs & how to avoid them
  • ID collisions: Track IDs in a single source; never reuse silently.
  • Null refs: Validate links from Class → SkillForce → ItemSetEffect.
  • Curve breaks: Watch for EXP gaps or negative thresholds.
Tip: keep a lightweight changelog next to your data files. Note the file, sheet, cell range, and intent for every edit.
Sample workflow snippet 
# Example: safe edit loop (pseudo)
1. Backup Character.edf → Character.edf.bak
2. Edit EXP sheet (Levels 10–20) by +4–6%
3. Repack → validate IDs and references
4. Reload server in staging → run scripted tests
5. Log diffs → playtest for 30 minutes across 3 classes
6. If stable: tag build, deploy to live during low traffic

Friendly guidance and safety

Small, steady changes beat big swings. Your players feel pacing shifts instantly—opt for clarity and control.

  • Backups: Always keep last-known-good and a test branch.
  • Diffs: Review changes with teammates before repack.
  • Rollbacks: Prepare a one-click revert path for live incidents.
